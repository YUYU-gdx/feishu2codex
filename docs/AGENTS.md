# Agent 工作指南

## 浏览器自动化工具使用优先级

在需要进行网页浏览、自动化操作时，请严格按照以下优先级选择工具：

### 1. browser-use (首选)
**优先使用场景：**
- 所有浏览器相关的任务
- 网页抓取和数据提取
- 自动化表单填写
- 网页截图
- 深度网页研究

**优势：**
- 支持自然语言控制
- 自动管理浏览器进程
- 内置视觉理解能力
- 无需手动配置

**使用示例：**
```
用户: "打开百度搜索 MCP 协议"
你应该: 调用 browser-use 的工具来完成
```

### 2. playwright (备选)
**仅在以下情况使用：**
- browser-use 调用失败或报错
- browser-use 不可用
- 需要更底层的浏览器控制

**使用示例：**
```
如果 browser-use 返回错误，则回退到 playwright
```

## 其他工具

### context7
**用途：** 查询最新的开发文档和 API 参考

**使用场景：**
- 查询技术文档
- 了解框架/库的最新用法
- 获取代码示例

### github
**用途：** 管理 GitHub 仓库、PR、Issues

**使用场景：**
- 查看仓库信息
- 创建/查看 Pull Request
- 管理 Issues

## 重要原则

1. **始终优先尝试 browser-use**
2. **只有在 browser-use 明确失败时才使用 playwright**
3. **不要同时使用两个浏览器工具**
4. **如果两个工具都失败，向用户说明情况**
